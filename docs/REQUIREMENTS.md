# linking-scrapbox 要件定義

## 概要

ScrapboxのエクスポートJSONを入力として、AIが各ページの関連分野を分析し、
間接リンクを挿入したJSONを出力するCLIツール。
**ユーザーが主体**となり、AIは候補の提案に徹する。

---

## コンセプト

- AIはリンクを「決定」しない。候補を「提案」するだけ
- すべての提案はユーザーが確認・承認・編集してから反映される
- リンクはページ同士を直接繋げず、「分野ページ」を経由した間接リンク構造にする

---

## CLIオプション

```bash
npx linking-scrapbox \
  --input export.json \       # Scrapboxエクスポートファイル（必須）
  --output output.json \      # インポート用出力ファイル（必須）
  --format rules.md \         # フォーマットルールファイル（任意）
  --tree-in tree.json \       # 前回のツリーファイル読み込み（任意）
  --tree-out tree.json        # 今回のツリーファイル保存（任意）
```

---

## 全体フロー

```
1. ファイル読み込み
   - export.json（必須）
   - rules.md（任意）: フォーマットルールをAIプロンプトに追加
   - tree.json（任意）: 前回承認済みツリーをAI提案の土台に使用

2. 分野ツリーの提案・確認
   - AIが全ページを分析し、木構造で分野ツリーを提案
   - ユーザーが確認・編集・承認

3. 分野ごとのページ紐づけ
   - 各分野に対してAIが該当ページ候補を提案
   - ユーザーが採用・編集・スキップ

4. 上位分野の提案
   - 複数の分野をまとめられる場合、上位分野の作成をAIが別途提案
   - ユーザーが承認した上位分野ページも出力JSONに含める

5. output.json を出力
   - 各ページの lines 先頭に分野リンク行を挿入
   - 新規作成された上位分野ページも含めて出力
   - Scrapboxインポート形式に準拠
```

---

## 機能詳細

### 機能1: フォーマットルールファイル（--format）

- `rules.md` または `rules.txt` を指定
- 中身はAIへの自然言語指示として、プロンプトにそのまま追加される
- 例:

```md
- リンク行は必ずページ先頭の1行目に挿入する
- 英語タイトルはそのまま使う（日本語に訳さない）
- 1ページあたりの分野リンクは最大3つまで
```

---

### 機能2: 分野ツリーの木構造表示・入出力

#### ツリーファイルフォーマット（tree.json）

```json
{
  "技術": {
    "プログラミング言語": {
      "Rust": {},
      "C": {
        "コンパイラ": {}
      },
      "Python": {}
    },
    "機械学習": {
      "自然言語処理": {}
    }
  },
  "イベント": {
    "ハッカソン": {}
  }
}
```

#### ステップ2: 分野ツリーの確認

```
[AI提案] 分野ツリー

技術
├── プログラミング言語
│   ├── Rust
│   ├── C
│   │   └── コンパイラ
│   └── Python
└── 機械学習
    └── 自然言語処理

イベント
└── ハッカソン

[e]編集  [y]承認  [r]再提案 >
```

- `e`: `$EDITOR` でツリーファイルを直接編集
- `y`: 承認して次のステップへ
- `r`: AIに再提案を依頼

---

### 機能3: 分野ごとのページ紐づけ（間接リンク）

#### ステップ3: ページ紐づけの確認

```
[分野] ハッカソン  (3/23)
紐づけ候補: [58ハッカソン] [ハッカソンの開発手法] [Googleハッカソン]

[y]全採用  [e]手動編集  [s]スキップ  [q]終了・保存 >
```

- 1ページが複数の分野に属する場合、それぞれの分野のステップで候補に登場する
- 挿入行は複数リンクをまとめて1行

```
# 58ハッカソン ページに挿入される行（複数分野に属する場合）
[ハッカソン] [個人開発]
```

---

### 機能4: 上位分野の提案

分野が複数まとめられそうな場合、AIが別途提案する。

```
[提案] 上位分野の作成  (1/3)

  [Rust] [C] [Python] [Go]
        ↓
  [プログラミング言語]  ← AIが命名提案

[y]作成  [n]スキップ  [e]名前を変更 >
```

上位分野ページ自体にも、さらに上の分野へのリンクが挿入される。

```
# プログラミング言語 ページに挿入される行
[技術]
```

---

## 出力フォーマット（output.json）

Scrapboxインポート形式に準拠。

```json
{
  "pages": [
    {
      "title": "58ハッカソン",
      "lines": [
        "[ハッカソン] [個人開発]",
        "...既存の内容..."
      ]
    },
    {
      "title": "ハッカソン",
      "lines": [
        "[イベント]",
        "...既存の内容..."
      ]
    },
    {
      "title": "プログラミング言語",
      "lines": [
        "[技術]"
      ]
    }
  ]
}
```

- 既存ページ: リンク行を先頭に追加（既にリンク行があれば上書き）
- 新規分野ページ: 上位分野へのリンク行のみ持つページとして新規生成

---

## 技術スタック（推奨）

- Node.js
- Anthropic SDK（`@anthropic-ai/sdk`）
- `inquirer` または `readline` でCLI対話
- `cli-tree` または自前実装でツリー表示

---

## 考慮事項

- **レート制限対策**: APIはバッチ処理（5〜10ページずつ）で呼び出す
- **冪等性**: 既にリンク行がある場合は上書き（先頭行を検出して判断）
- **自己参照除外**: 自分自身のタイトルは関連候補に含めない
- **中断・再開**: `--tree-in` で途中のツリーを再利用することで実質的に再開可能
